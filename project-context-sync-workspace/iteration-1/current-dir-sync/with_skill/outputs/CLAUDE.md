# 小红书热点内容工厂 - 项目上下文

## 项目概述

**项目名称**: 小红书热点内容工厂 (XiaoHongShu Content Factory)
**项目类型**: AI驱动的内容生产工具
**当前状态**: 规划阶段 (PRD已完成)
**文档版本**: v1.0
**创建日期**: 2026-02-24

## 产品定位

面向小红书内容创作者和运营团队的AI驱动内容生产工具，以视频笔记为主要输出形式，提供从热点抓取、内容生成、视频制作到发布管理的一站式解决方案。

### 核心价值

将单条笔记的生产时间从传统的 **3-6小时** 缩短至 **< 10分钟**:

| 维度 | 传统方式 | 本产品 |
|------|----------|--------|
| 热点发现 | 手动浏览，30-60分钟 | 关键词订阅，实时推送，< 1分钟 |
| 文案生成 | 人工撰写，1-2小时 | AI生成 + 人工微调，< 5分钟 |
| 视频制作 | 剪辑软件操作，1-3小时 | 自动生成，< 3分钟 |
| 发布管理 | 手动逐条发布 | 队列管理 + 定时发布 |

## 技术架构

### 技术栈

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端框架 | React + TypeScript | 类型安全、生态成熟 |
| UI组件库 | Ant Design | 企业级组件、开箱即用 |
| 后端框架 | Python FastAPI | 异步支持、自动API文档、类型校验 |
| 任务队列 | Celery + Redis | 异步任务处理、定时任务 |
| 文案AI | DeepSeek V3 | 中文能力强、性价比高、API稳定 |
| 图片AI | Flux (Replicate) | 生成质量高、API简单、按量计费 |
| 视频生成 | FFmpeg + Python | 开源、功能强大、可定制 |
| 数据库 | SQLite → PostgreSQL | 本地开发零配置、上云后平滑迁移 |

## 核心功能模块

### 1. 热点发现 (Trend Discovery)
- **关键词管理**: 添加/编辑/删除关键词，支持领域分类
- **热点抓取引擎**: 自动抓取小红书热点话题
- **热点排序与筛选**: 按热度/时间排序，支持领域筛选
- **热点详情查看**: 查看话题详情、相关笔记、数据趋势

### 2. 内容生成 (Content Generation)
- **AI文案生成**: 基于DeepSeek V3生成标题、正文、标签
- **AI封面图生成**: 基于Flux生成封面图
- **文字动画视频生成**: 基于FFmpeg生成30-60秒视频
- **内容预览与编辑**: 实时预览，支持文案/封面/视频编辑

### 3. 发布管理 (Publish Management)
- **发布队列**: 待发布内容队列管理
- **定时发布**: 设置发布时间，自动发布
- **发布历史**: 查看已发布内容及数据
- **发布状态监控**: 实时监控发布状态

### 4. 系统设置 (System Settings)
- **API Key管理**: 配置DeepSeek、Replicate API Key
- **定时任务配置**: 设置热点抓取频率
- **默认参数配置**: 视频时长、音乐风格等

### 安全性
- API Key使用AES-256加密存储
- 日志中不记录API Key、Cookie等敏感信息
- 小红书Cookie定期更新机制

### 性能要求
- 文案生成时间: < 15秒
- 封面图生成时间: < 60秒
- 视频生成时间: < 180秒
- API调用超时: 60秒
- 数据库查询: < 200ms

### 扩展性
- 数据库支持从SQLite平滑迁移到PostgreSQL
- 封面图生成模型可配置（Flux / DALL-E / Midjourney API）
- 架构预留其他平台（抖音、微博）的扩展接口
